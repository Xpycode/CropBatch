### Bug Fix — Image flash on thumbnail switch
- **Root cause:** `scaledImageView` downscale branch used `Image(nsImage:)` without `.resizable()`. On cache miss (every image switch), full-res image rendered at native pixel dimensions for one frame.
- **Fix:** Added `.resizable().aspectRatio(contentMode: .fill)` to both branches + guarded content on `viewSize > 0`
- Added to cookbook: "Image Cache Miss Flash" pattern

### Bug Fix — Blur export coordinates shifted
- **Root cause:** Pipeline applied blur BEFORE crop (Blur → Transform → Crop). Blur regions stored in original-image space were baked at absolute coordinates, then crop shifted the origin — blur appeared offset by `cropTop` pixels.
- **Fix:** Reordered pipeline to Transform → Crop → Blur. Blur regions now remapped to post-crop space via `applyingTransform` + `relativeToCrop`. Regions outside crop area filtered out.
- Existing `regionsForExport` / `relativeToCrop` infrastructure was already in place — just needed to be used.

### Bug Fix — Corner radius doesn't auto-set PNG
- **Root cause:** Enabling corner radius requires PNG for transparency, but format picker stayed on user's selection. Pipeline silently overrode to PNG at export, confusing the user.
- **Fix:** `onChange` on `cornerRadiusEnabled` saves current format and switches to `.png`. Toggle off restores previous format. Format picker disabled with caption when locked.
