### Bug Fix — Image flash on thumbnail switch
- **Symptom:** When clicking next thumbnail, image briefly fills entire left pane at full resolution before settling to correct scale
- **Root cause:** `scaledImageView` else branch (downscale path) used `Image(nsImage:)` without `.resizable()`. On cache miss (every image switch), the full-res fallback image rendered at native pixel dimensions — `.frame()` only affects layout, not intrinsic image size
- **Secondary cause:** `viewSize` initialized to `.zero`, `currentScale` fell back to `1.0` on first render before GeometryReader reported size
- **Fix:** Added `.resizable().aspectRatio(contentMode: .fill)` to the else branch + guarded content rendering on `viewSize > 0`
- **Added to cookbook:** "Image Cache Miss Flash" pattern in SwiftUI Performance section
