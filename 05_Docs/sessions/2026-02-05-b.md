### Root Cause
`ENABLE_APP_SANDBOX = YES` was set in Xcode project build settings. This injects the sandbox entitlement at build time even though the `.entitlements` file was empty. The sandbox blocked all outgoing network connections — Sparkle couldn't fetch `appcast.xml`.

The project intended to be unsandboxed (per CLAUDE.md), but this setting was never changed.

### Files Changed
| File | Changes |
|------|---------|
| `project.pbxproj` | `ENABLE_APP_SANDBOX` → `NO`, removed `ENABLE_USER_SELECTED_FILES` (both configs) |

## Decisions
- Disable sandbox entirely rather than adding network entitlement — app doesn't need sandbox
- Hardened Runtime kept (required for notarization, doesn't block networking)
- v1.2 users can't auto-update (sandbox blocks it) — must manually download v1.3
- Adopted build numbering scheme: 130-139 for v1.3, 140-149 for v1.4, etc.
- `sparkle:version` must always match `CFBundleVersion` — never use arbitrary numbers

## Lessons Learned
1. App Sandbox blocks networking without `com.apple.security.network.client` — always test update check after enabling sandbox
2. `sparkle:version` is the ONLY value Sparkle compares — marketing version is display-only
3. `sparkle:version` must match `CFBundleVersion` of the distributed binary, or Sparkle will loop or offer downgrades
